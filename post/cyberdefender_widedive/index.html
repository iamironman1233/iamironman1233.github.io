Lưu ý: Ở mỗi bài sẽ có tên file cần sử dụng ở trước

dhcp.pcapng

(DHCP có thể hiểu là Dynamic Host Configuration Protocol)

What IP address is requested by the client?

Tiến hành mở file lên bằng wireshark và tìm địa chỉ IP được yêu cầu từ client

Đọc trang DHCP và Display Filter Reference: Bootstrap Protocol và lọc bootp.option.dhcp == 3 ta được:

–> 192.168.2.244

What is the transaction ID for the DHCP release?

Lọc bootp.option.dhcp để gọi các gói DHCP ra, sau đó để ý kĩ phần Info ta được điều cần tìm

–> 0x9f8fa557

What is the MAC address of the client?

Sử dụng lại phần của mục 1 - Request

–> 00:0c:29:82:f5:94

dns.pcapng

What is the response for the lookup for flag.fruitinc.xyz?

Lọc dns.resp.name == "flag.fruitinc.xyz"

–> ACOOLDNSFLAG

Which root server responds to the query? Hostname.

Check trong trang DNS Root Servers thì có thể thấy xuất hiện địa chỉ 192.203.230.10

và nó tương ứng với e.root-servers.net trên trang vừa rồi

–> e.root-servers.net

smb.pcapng

What is the path of the file that is opened? (tìm đường dẫn của file được mở)

Sử dụng List of SMB2 Opcodes ta có thể thấy các yêu cầu đọc file được ký hiệu bằng opcode 0x08.

Chúng ta sẽ áp dụng lọc smb2.cmd == 8 và được

–> HelloWorld\TradeSecrets.txt

What is the hex status code when the user SAMBA\jtomato logs in? (trạng thái mã hex khi người dùng SAMBA\jtomato đăng nhập là gì?)?

Sử dụng List of SMB2 Opcodes ta có thể thấy các yêu cầu đọc file được ký hiệu bằng opcode 0x01.

Lọc smb2.cmd == 1 ta được:

Ở gói 75 có nhắc đến người dùng SAMBA\jtomato đăng nhập và ngay gói 76 đã có lỗi đăng nhập, check gói 76 thì ta được:

–> 0xc000006d

What is the tree that is being browsed?

Sử dụng List of SMB2 Opcodes ta lọc bằng smb2.cmd == 3:

Mặc dù hiển thị có 2 yêu cầu Tree Connect (IPC$ và public), nhưng chia sẻ cuối cùng được duyệt là public

–> \\192.168.2.10\public
